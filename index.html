Web 前端开发模式演变历史和趋势

主讲人：钟远维

2016年9月25日

  • 后端为主的时代
  • 前端获取自主权的MV*时代
  • 前端工业化生产的到来
后端为主的时代
初始阶段 —— 简单明快的早期时代

特点：

不分前后端，经常 3-5 人搞定所有开发。页面由 JSP、PHP 等工程师在服务端生成，浏览器负责展现。基本上是服务端给什么浏览器就展现什么，展现的控制在 Web Server 层。

问题：

1、Service 越来越多，调用关系变复杂，前端搭建本地环境不再是一件简单的事。

2、JSP 等代码的可维护性越来越差。

后端为主的时代
后端的 MVC 时代

为了降低复杂度，以后端为出发点，有了 Web Server 层的架构升级，比如 Structs、Spring MVC 等，这是后端的 MVC 时代。

优点：

分工明确，代码维护性变得更好了。

View 层更简单干脆，还可以选择 Velocity、Freemaker 等模板。

问题：

1、前端开发重度依赖开发环境。

2、前后端职责依旧纠缠不清。

后端为主的时代
Ajax 带来的 SPA 时代

特点：

前后端的交互不再只是完整的 HTML ，还可以是 HTML 片段， JSON 数据。

出现了好多工具类库， jQuery / Prototype / Dojo / underscore。

前端获取自主权的MV*时代
前端框架百花齐放

除了 Backbone ，还涌现出 EmberJS，KnockoutJS，AngularJS，Vue.js，ReactJS等MV*框架。

特点：

前后端职责很清晰。

前端开发的复杂度可控。

部署相对独立，产品体验可以快速改进。

前端工业化生产
基于 Node 的前端开发

开发工具：

构建工具：grunt、 browserify、 gulp、 webpack、 jspm。

包管理工具：components、 bower、 spm、 npm。

测试、监控：QUnit、 Selenium、 Jshint、 WebDriverTest。

全栈开发模式：

对前端来说，两个 UI 层各司其职：

1、Front-end UI layer 处理浏览器层的展现逻辑。通过 CSS 渲染样式，通过 JavaScript 添加交互功能，HTML 的生成也可以放在这层，具体看应用场景。

2、Back-end UI layer 处理路由、模板、数据获取、cookie 等。
